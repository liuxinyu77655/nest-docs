# 107. 如何拿到服务器 CPU、内存、磁盘状态

在很多系统中，会有展示服务器信息的页面。

比如 CPU、内存、磁盘等：

![](https://vercel-nestjs.oss-cn-beijing.aliyuncs.com/nest-docs/image/107-1.png)

这是怎么实现的呢？

其实通过 node 的原生 api 就可以做到。

我们还是用 Nest 写个接口来返回这些信息：

```
nest new server-status
```

![](https://vercel-nestjs.oss-cn-beijing.aliyuncs.com/nest-docs/image/107-2.png)

在 AppController 加个接口，用 os.cpus 拿到 cpu 的信息：
javascript

```
import{ Controller, Get }from'@nestjs/common';import{ AppService }from'./app.service';import*asosfrom'os';@Controller()exportclassAppController{constructor(privatereadonlyappService:AppService) {}@Get()getHello():string{returnthis.appService.getHello();}@Get('status')status() {returnos.cpus();}}
```

把服务跑起来：

```
npm run start:dev
```

![](https://vercel-nestjs.oss-cn-beijing.aliyuncs.com/nest-docs/image/107-3.png)

浏览器访问下：

![](https://vercel-nestjs.oss-cn-beijing.aliyuncs.com/nest-docs/image/107-4.png)

返回的数组元素个数就是 cpu 数。

那具体的属性是什么意思呢？

![](https://vercel-nestjs.oss-cn-beijing.aliyuncs.com/nest-docs/image/107-5.png)

可以看到，times.user、times.sys、times.idle 分别代表用户代码占用的 cpu 时间、系统代码占用的 cpu 时间，空闲的 cpu 时间：

![](https://vercel-nestjs.oss-cn-beijing.aliyuncs.com/nest-docs/image/107-6.png)

基于这些就能算出 cpu 的使用率、空置率来。
javascript

```
@Get('status')status() {constcpus=os.cpus();constcpuInfo=cpus.reduce((info,cpu)=>{info.cpuNum+=1;info.user+=cpu.times.user;info.sys+=cpu.times.sys;info.idle+=cpu.times.idle;info.total+=cpu.times.user+cpu.times.sys+cpu.times.idle;returninfo;},{ user:, sys:, idle:, total:, cpuNum:},);constcpu={cpuNum: cpuInfo.cpuNum,sys: ((cpuInfo.sys/cpuInfo.total)*100).toFixed(),used: ((cpuInfo.user/cpuInfo.total)*100).toFixed(),free: ((cpuInfo.idle/cpuInfo.total)*100).toFixed(),};returncpu;}
```

用 reduce 方法累加 cpu 的数量、user、sys、idle 以及总的 cpu 时间。

然后 cpu 的系统使用率就是 sys/total，用户使用率是 user/total 而空置率就是 idle/total。

![](https://vercel-nestjs.oss-cn-beijing.aliyuncs.com/nest-docs/image/107-7.png)

对应的就是这部分信息：

![](https://vercel-nestjs.oss-cn-beijing.aliyuncs.com/nest-docs/image/107-8.png)

然后继续拿内存信息：
javascript

```
import{ Controller, Get }from'@nestjs/common';import{ AppService }from'./app.service';import*asosfrom'os';@Controller()exportclassAppController{constructor(privatereadonlyappService:AppService) {}@Get()getHello():string{returnthis.appService.getHello();}@Get('status')status() {return{cpu:this.getCpuInfo(),mem:this.getMemInfo()}}getMemInfo() {consttotalMemory=os.totalmem();constfreeMemory=os.freemem();constusedMemory=totalMemory-freeMemory;constmemoryUsagePercentage=(((totalMemory-freeMemory)/totalMemory)*100).toFixed();constmem={total: totalMemory,used: usedMemory,free: freeMemory,usage: memoryUsagePercentage,};returnmem;}getCpuInfo() {constcpus=os.cpus();constcpuInfo=cpus.reduce((info,cpu)=>{info.cpuNum+=1;info.user+=cpu.times.user;info.sys+=cpu.times.sys;info.idle+=cpu.times.idle;info.total+=cpu.times.user+cpu.times.sys+cpu.times.idle;returninfo;},{ user:, sys:, idle:, total:, cpuNum:},);constcpu={cpuNum: cpuInfo.cpuNum,sys: ((cpuInfo.sys/cpuInfo.total)*100).toFixed(),used: ((cpuInfo.user/cpuInfo.total)*100).toFixed(),free: ((cpuInfo.idle/cpuInfo.total)*100).toFixed(),};returncpu;}}
```

os.totalmem 是总内存，os.freemem 是空闲内存，那差值就是已使用的内存。

这样也可以求出内存使用率。

![](https://vercel-nestjs.oss-cn-beijing.aliyuncs.com/nest-docs/image/107-9.png)
